隐私政策

本政策说明随页浏览器扩展如何在用户主动使用相关功能时处理、保存和保护数据。

生效日期：2026 年 7 月 21 日最近更新：2026 年 7 月 23 日
本机处理请求记录和工具输入默认仅在用户设备中处理。
不设开发者服务器随页不会把捕获的数据上传给开发者或分析平台。
由用户控制用户决定何时开始、停止、清空、复制或导出数据。

1. 适用范围

本政策适用于“随页”浏览器扩展。随页的单一用途是为用户提供跟随当前网页的本地调试与效率工具，包括网络请求捕获、请求覆写、测试表单智能填充、开发转换、MFA 动态验证码、二维码生成和内容差异比较。

2. 随页处理的数据

在用户主动开启并保持“捕获流量”功能期间，随页可能处理并在本机保存以下信息：

  • 浏览器标签页的 URL、网页标题、标签页标识和访问时间。
  • 网络请求 URL、方法、类型、状态码、耗时、错误信息及重定向信息。
  • 请求与响应 Header、Request Body 和 Response Body。
  • 用户创建的请求覆写规则、扩展设置和数据保留偏好。
  • 用户主动输入到内容转换、二维码或差异比较工具中的文本。
  • 用户主动添加到 MFA 验证器的服务名称、账户名称、Base32 密钥和验证码参数。
  • 用户主动使用智能填充时，当前网页表单字段的名称、标签、占位文字和自动填充属性会在网页本地临时用于识别字段类型；这些字段描述不会被保存。
敏感数据提示：完整请求或响应内容可能包含个人身份信息、身份验证凭据、个人通讯、位置、健康或财务信息。随页不会主动识别或分析这些类别，但原始网络内容可能包含它们。请仅在您有权调试的网页上开启捕获，并及时清理不再需要的记录。

3. 数据用途

上述数据仅用于用户可见的扩展功能：

  • 在侧边栏中显示、筛选和检查网络请求历史。
  • 展示请求与响应详情、错误原因、耗时和正文。
  • 按照用户创建并启用的规则覆写请求 Header 或 Query 参数。
  • 根据用户操作复制请求信息、生成 HAR 文件或执行本地内容处理。
  • 根据用户保存的 MFA 密钥在本机计算并显示动态验证码。
  • 在用户点击“填充当前网页”后，将本机生成的虚构测试身份或支付平台官方沙箱测试卡写入识别到的表单字段；随页不会自动提交表单。

随页不会将这些数据用于广告、用户画像、信用评估或与上述功能无关的目的。

4. 本机存储与保留

请求历史通过浏览器提供的 IndexedDB 保存在用户设备中；覆写规则、捕获状态、偏好设置以及用户主动添加的 MFA 账户和密钥保存在本地扩展存储中。随页不使用云端同步保存请求内容或 MFA 数据。

请求缓存受数量、单条正文、总容量和保留时间限制，默认自动清理超过一小时的历史记录。用户可以在扩展中查看当前缓存占用并清空当前网页或全部网页的请求记录，也可以通过卸载扩展移除扩展保存的数据。

5. 数据传输、分享与出售

随页不会把捕获记录、工具输入、覆写规则或 MFA 数据发送到开发者控制的服务器，也不会出售、出租或分享给第三方。

当用户主动复制或导出数据时，相关内容会进入用户设备的剪贴板或下载文件；其后续保管和分享由用户自行控制。用户创建的覆写规则可能改变发送给目标网站的请求内容，该请求仍由浏览器发送至用户正在访问的目标网站。

6. 浏览器权限

  • sidePanel在浏览器侧边栏中显示随页的主要界面。
  • storage在本机保存请求历史、覆写规则和用户偏好。
  • alarms定期执行用户设置的本机请求缓存清理任务。
  • declarativeNetRequest执行用户明确创建并启用的 Header 与 Query 覆写规则。
  • debugger在用户开启捕获后读取请求生命周期、请求与响应 Header 以及正文；停止捕获后断开连接。
  • tabs按照顶层网页标签整理请求，并跟随标签页切换、新建、刷新或关闭状态。
  • scripting仅在用户主动使用 MFA 二维码框选或智能填充时，向当前网页注入临时选择层或字段识别脚本；框选完成后移除选择层，填充脚本只写入用户选择的测试资料且不会提交表单。
  • clipboardWrite仅在用户点击复制按钮时写入请求信息、工具结果或二维码图片。
  • <all_urls>让用户创建的覆写规则能够应用到安装前无法预知的网站，并允许用户主动在当前 HTTP/HTTPS 网页中框选 MFA 二维码或填充测试表单。

7. 用户选择与控制

  • 用户可以随时开始或停止网络捕获。
  • 用户可以停用或删除任意覆写规则。
  • 用户可以清空单个网页或全部网页的请求记录。
  • 用户可以删除单个 MFA 账户或一键清空全部 MFA 账户。
  • 用户可以通过浏览器扩展管理页面撤销权限或卸载随页。

8. 数据安全

随页不提供远程账户、遥测或分析服务，扩展代码和依赖均包含在安装包中。请求记录和 MFA 密钥都可能包含敏感信息，请保护设备和浏览器配置文件，并避免在共享设备上长期保留请求记录、导出的 HAR 文件或 MFA 账户。

9. Chrome Web Store Limited Use

随页对通过 Chrome API 获得的信息的使用，将遵守 Chrome Web Store 用户数据政策，包括 Limited Use 要求。相关数据仅用于提供或改进随页向用户展示的核心功能。

10. 政策更新与联系

如果随页的数据处理方式发生变化，本政策将同步更新，并在需要时于扩展界面中向用户提供显著说明。用户可以通过随页在 Chrome 网上应用店商品详情页提供的支持渠道联系开发者。

有关 Chrome Web Store 用户数据规则，请参阅 Chrome Web Store 用户数据政策说明。
